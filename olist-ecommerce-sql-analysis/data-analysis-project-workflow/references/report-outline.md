# Business Analysis Report Outline

## Recommended structure

1. Project background and analysis objective
2. Data source, time range, and processing rules
3. Core findings summary
4. Trend and structural analysis
5. Customer or user analysis
6. Product, region, content, or seller analysis
7. Experience, risk, or operational efficiency analysis
8. Business recommendations
9. Project summary and skill coverage

## Writing pattern for each section

```text
Question: What business issue is being examined?
Evidence: What metric or chart supports the finding?
Finding: What pattern, change, or exception appears?
Action: What should the business do next?
```

## Report quality rules

- Put the data limitation near the relevant analysis, not only in an appendix.
- Use exact values only where they support a decision; do not fill the report with raw query output.
- Use tables for comparable metrics and prose for explanation and recommendations.
- Keep each chart close to the paragraph that interprets it.
- Ensure every recommendation can be traced back to a reported result.
